# PR Template

> This file defines the PR body format.
> @ns-dev-coder uses this when creating PRs.
> Customize it for your project's PR conventions.

## Template

```markdown
## Summary
Closes #<number>

- <bullet 1: what was done>
- <bullet 2>

## Plan
`docs/plans/issue-<number>-<slug>-<date>.md`

## Changes
- <file/area changed and why>

## Verification
- [x] Typecheck passes
- [x] Tests pass
- [ ] E2E tests (pending @ns-dev-tester)

## Assumptions / Deviations from Plan
- <any decisions made that weren't in the plan>
```
