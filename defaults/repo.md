## Branch Naming

Pattern: `issue-{number}-{slug}`

Example: `issue-27-add-user-settings`

The slug is 2-3 words from the issue title, kebab-case.

## Commit Messages

Pattern: `{type}(issue-{number}): {description}`

Where `{type}` is:
- `feat` — new feature
- `fix` — bug fix

## Labels

These labels drive the pipeline state machine. All are required.

| Label | Color | Description |
|-------|-------|-------------|
| `status:planning` | `#1d76db` | Issue is being planned |
| `status:plan-review` | `#5319e7` | Plan ready for review |
| `status:plan-revising` | `#fbca04` | Plan needs revision |
| `status:approved` | `#0e8a16` | Plan approved, ready for implementation |
| `status:code-review` | `#5319e7` | PR ready for code review |
| `status:code-revising` | `#fbca04` | Code needs revision |
| `status:testing` | `#1d76db` | PR ready for testing |
| `status:ready-to-merge` | `#0e8a16` | All checks passed, ready for human merge |
| `status:blocked` | `#d93f0b` | Blocked, needs human intervention |
| `status:needs-info` | `#d93f0b` | Issue needs clarification |
